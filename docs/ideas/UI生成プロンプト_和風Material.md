# UI生成プロンプト（和風 × Material Design）

合体漢字ガチャの画面UIを生成AIに作らせるためのプロンプト集。
そのままコピーして画像生成AI／コード生成AIに渡せる。

- **A. マスタープロンプト**：世界観・デザインシステムの共通定義（最初に1回読み込ませる）
- **B. 画面別プロンプト**：A を前提に各画面を作らせる
- **C. ガチャ演出プロンプト**：おみくじ巻物リビールの専用指示
- **D. 画像生成AI向け（英語・ビジュアル用）**：モックや素材生成用

## A. マスタープロンプト（共通デザインシステム）

```
あなたは和風×Google Material Designに精通したUIデザイナー兼フロントエンド実装者です。
「合体漢字ガチャ」というWebゲームの画面UIを作成してください。

# ゲームの世界観
- 漢字のパーツ（部首・つくり）をガチャで集め、組み合わせて漢字を「合体」させる学習ゲーム
- 1プレイ1〜2分のカジュアルなスナックゲーム
- 全体に「和紙・墨・朱印・巻物」を思わせる和の質感をまとう

# デザインの大原則（必ず守る）
1. Google Material Design 3 に遵守する
   - カラーは Material のデザイントークン（--md-sys-color-*）として定義し、全画面で参照する
   - フォントサイズ・行間は Material の Type Scale（Display / Headline / Title / Body / Label）に従う
   - 角丸（shape-corner small/medium/large/full）・エレベーション（影）も Material のスケールに揃える
2. 和風カラーパレットを Material のロールにマッピングする
   - Primary   = 朱色（赤系・例 #C0392B）
   - Secondary = 藍色（青系・例 #2C3E6B）
   - Tertiary  = 金色（アクセント・レアリティ・達成表示・例 #B8860B / #D4AF37）
   - Surface   = 和紙色（オフホワイト／ベージュ・例 #F5F1E6）
   - On-Surface= 墨色（ほぼ黒・例 #2B2B2B）
3. タイポグラフィ
   - 漢字パーツ・見出し・ボタンの文字は「墨で描いたような筆文字／明朝系」にする
   - 筆文字Webフォントは自前ホスト前提（外部CDN・Google Fontsは使わない＝CSP配慮）
   - フォントが無い場合のフォールバックに明朝系システムフォントスタックを必ず指定する
   - 数値（スコア・残数・時間・収集率）は等幅数字（tabular-nums）で表示する

# アクセシビリティ・実装制約
- prefers-reduced-motion 設定時は、装飾アニメーションを即時表示（瞬間切替）にフォールバックする
- 装飾用のオーバーレイ・エフェクトには pointer-events: none を付け、操作（ボタン・カード選択）を妨げない
- 既存のテスト用属性（data-testid・aria-*・role）やボタンの表示文言は変更しない前提で、
  見た目（CSS・配色・字形・装飾）のみを再デザインする
- 実装する場合は Svelte 5（runes構文）のコンポーネントとして出力する

# 出力形式
- まずデザイントークン一覧（CSS変数）を提示し、その後に各コンポーネントを出力する
- 各ファイル冒頭に役割を1行コメントで添える
```

## B. 画面別プロンプト（A を前提に使う）

### B-1. ホーム画面
```
[A を適用]
レベル選択（やさしい/ふつう/むずかしい）・今日のお題・タイムアタック・図鑑/Aboutへの導線を持つ
トップ画面を作成してください。
- レベル札は「お神籤の番号札／短冊」風の和紙カード。難易度が上がるほどアクセントを藍→朱→金に変える
- 各札にベストスコアを等幅数字で併記
- 「今日のお題」は金枠＋落款バッジで特別感を出す
- 1タップでゲーム開始できる導線を崩さない
```

### B-2. プレイ画面（メイン）
```
[A を適用]
ガチャ・合体・ヒント・捨てて引き直しを行うメイン画面を作成してください。
- 上部にスコア／コンボ倍率／ガチャ残（またはタイムアタックの残り時間バー）を常時表示
- 手札の漢字パーツは「かるた札」風カード。漢字本体は墨筆フォントで大きく、右上にレアリティの落款印
- 選択中は枠を朱色、ヒント中は金色のリングで強調
- ボタンは Material の filled（ガチャ・合体＝主要アクション）／outlined（ヒント・捨てる・やめる）
- ガチャ実行時は「C. ガチャ演出」の巻物リビールを表示（操作を妨げない装飾オーバーレイ）
```

### B-3. 結果画面
```
[A を適用]
プレイ終了後の結果画面を作成してください。
- 称号（ランク）を巻物／賞状風の枠に墨筆フォントで大きく表示（最も目立たせる）
- 新記録は金色のリボン／落款で祝祭感を出す。スコアは等幅数字でアクセント色
- 作った漢字・新発見漢字はかるた札風の小カードで一覧（新発見は金縁）
- もう一回（filled）／シェア・ホーム（outlined）
```

### B-4. 図鑑画面
```
[A を適用]
発見した漢字の一覧と収集率を表示する図鑑画面を作成してください。
- 古書／図鑑のイメージ。背景に和紙テクスチャ
- 収集率バーは金色グラデーション、収集数は等幅数字で大きく
- 発見済みは札風カード（墨字＋読み・意味）、未発見は墨で塗られた札の裏面風に「？」
```

### B-5. About画面
```
[A を適用]
データ出典・ライセンス・フォントクレジット・データ保存の注意を表示する静的画面を作成してください。
- 各セクションを和紙風カード（薄い罫線・角丸medium）に。見出しは墨筆フォント
- 演出は最小限にし、可読性とテキストの正確さを最優先
```

## C. ガチャ演出プロンプト（おみくじ巻物リビール）

```
[A を適用]
ガチャの獲得演出を「おみくじ」風に作成してください。
- ガチャボタンを押すと、画面中央に巻物（絵巻物）が出現する
- 巻物が左右に開くアニメーションで展開し、開いた紙面の中央に
  「引いた漢字パーツ」が墨で一筆ずつ書かれていく（筆順アニメーション）
  - 実装：SVGパスの stroke-dashoffset を1画ずつ 0 に遷移させる筆順描画
  - 筆順データが無い文字は、明朝フォントのグリフを上→下に clip-path でワイプして
    「墨で書いている」風にフォールバックする
- パーツのレアリティに応じて巻物の落款（印影）の色を変える（高レアほど金・朱を強める）
- 描画完了後は自動的に巻物が閉じて消える
- この演出は完全に装飾用のオーバーレイ（pointer-events: none）とし、
  カード選択や各ボタン操作を一切ブロックしない
- prefers-reduced-motion 時は筆順アニメを省略し、完成形を即時表示する
- スクリーンリーダー向けに「〇〇を手に入れた」を aria-live="polite" で通知する
```

## 補足メモ
- 動的に出る漢字パーツは種類が多く、筆文字Webフォントのサブセットに収めきれない。
  そのため「見出し・ボタンは自前ホスト筆文字／漢字パーツは明朝系＋筆順アニメ」で墨筆感を出すのが現実的。
- 外部フォントCDNを使わない（CSP `default-src 'self'` 準拠）。フォントは subset woff2 を自前ホストする。
- 関連：[画面デザイン参考](./漢字合体ガチャ_企画コンセプト整理書.md) §11 ほか。
