# 学習機能アイデア（学年・対象別の漢字学習用途）

じっくり（達成型・山札）レベルを、**対象ユーザーの漢字学習に使える**ように強化するアイデア集。
タイムアタック（常用・実力勝負）とは別に、じっくりを「遊びながら覚える／復習する」体験にする方向。

## 0. 対象ユーザー別レベル設計（確定方針）
モード表記を学習対象が直感的に分かる名称に変更する。**小学生モードは学年で細分**する。

| モード | 対象ユーザー | 対象漢字 | 字数 |
|---|---|---|---|
| **小学生モード**（旧「やさしい」） | 小学生 | 小学校で習う漢字（grade 1–6） | 1,006字 |
| **大人モード**（旧「ふつう」） | 大人 | **小学生漢字以外のすべて**（KANJIDIC2 grade 8＝中学以降の常用） | 1,130字 |
| タイムアタック（変更なし） | 実力勝負 | 常用漢字すべて | 2,136字 |

### 小学生モード（学年で選ぶ7モード）
小学生モードを選んだら、**学年を選択**する。各学年＋全学年の**7モード**：

| サブモード | 対象 | 字数 |
|---|---|---|
| 小1 | 1年で習う漢字 | 80 |
| 小2 | 2年 | 160 |
| 小3 | 3年 | 200 |
| 小4 | 4年 | 200 |
| 小5 | 5年 | 185 |
| 小6 | 6年 | 181 |
| 全学年 | 小1〜小6すべて | 1,006 |

- 各サブモードは達成型（その範囲の漢字を分解した山札・収集率で評価）。
- 学年別にできることで「いま習っている学年だけ練習」でき、学校教材と並走できる。

### 大人モード
**小学生漢字以外のすべての漢字**（grade 8＝中学以降の常用・1,130字）を対象とする達成型。
学び直し・漢字検定対策・やり込み層向け。
（注：中学を中1〜中3に割る公式基準は KANJIDIC2 に無い＝grade 8 は単一バケツのため、
大人モードは中学以降をまとめて1つにする。）

### タイムアタック（既存・変更なし）
常用漢字すべて（2,136字）を時間制で競う。**スコア**で評価する点が、収集で評価する
小学生／大人モードと異なる別モードとして両立する。

### 実装メモ（学年分割の前提）
- 学年分割に必要な **grade はソース（`data-sources/kanjidic2-joyo.json`）にはあるが、
  配信用 `public/data/kanji.json` には含まれていない**。学年フィルタを使うには次が必要：
  1. データ生成（`generate.ts`）で `grade` を出力に含め、`kanji.json` を再生成
  2. `KanjiEntry` 型に `grade` を追加し、`DictionaryRepository` の山札構築
     （`buildDeck`/`deckTargetKanji`）に **grade フィルタ**を追加
  3. Home に「小学生モード → 学年（7択）」の2段選択 UI を追加
  4. 大人モードは「小学生（grade 1–6）以外」＝ grade ∉ {1..6} の山札に変更
- 既存の `Level`（elementary/juniorhigh/joyo）との関係：小学生モード＝elementary、
  大人モード＝juniorhigh＋joyo（grade 8 全体）。タイムアタックは常用全体（joyo スコープ）。

## 0.5 ゲーム進行ルール（じっくりモード＝小学生／大人。タイムアタックは対象外）

タイムアタック以外（達成型）に、**出題数の選択**と**重複禁止**・**引き直しの山札返却**を導入する。

### 開始フロー（2段選択）
1. **モードを選ぶ**（小学生モード → さらに学年7択／大人モード）
2. **出題数を選ぶ**（プリセット）
3. **ゲーム開始**

### 出題数 ＝ 「N字の山札」
- 選んだモードの対象漢字から **N字をランダムに選出**し、その **N字の部品だけで山札を構築**する。
- 目標：その **N字すべてを完成**させる（収集率 = 作れた数 / N）。
- 出題数は**プリセットから選ぶ**：例 **5 / 10 / 20 / 50 / 全部**
  （「全部」は対象範囲の完成可能字すべて。対象総数が N 未満ならその総数に丸める）。

### 制限：同じ漢字は作れない（重複禁止）
- 一度作った漢字は**もう一度作れない**（出題は N 字すべて異なる字）。
- すでに作成済みの漢字を作ろうとした合体は **無効（ノーペナルティ）**：
  「もう作ったよ」と表示し、**部品は消費せず・コンボも維持**（ミス扱いにしない）。

### 引き直し：パーツを山札に戻す
- 「捨てて引き直す」は、捨てた部品を**山札に戻してシャッフルし、別の1枚を引く**
  （部品を場から失わない）。
- これにより**N字すべてを完成できる状態が常に保たれる**（必要な部品が引き直しで永久に失われない）。

### 終了条件
- **N字すべて完成でクリア（達成）**。
- もしくは（重複禁止で手詰まり等の場合）山札・手札で**新たに作れる未完成字が無くなったら終了**し、
  そこまでの収集率で評価する。

### 実装メモ（現行 #63 からの差分）
- `buildDeck` を「対象から N 字サンプリング → その部品で山札構築」に拡張（出題数 N と RNG 注入）。
  `targetTotal = N`。
- `combine`：成立しても **awarded が作成済みなら無効（no-op・ノーペナルティ）**にする分岐を追加。
- `discardAndDraw`：現行は部品を消費するが、**捨てた部品を山札へ戻してシャッフル→1枚引く**に変更。
- 終了判定：N 字完成、または「未完成の到達可能字が無い」で終了。
- Home：モード選択 → 出題数（プリセット）選択 → 開始 の2段 UI。
- タイムアタックは本ルールの対象外（常用・時間制・スコア評価のまま）。

下記の学習支援機能（1〜5）は、上記いずれのモード（小学生の各学年／大人）でも共通して有効。

実現しやすさの目安：
- ◎ 既存データ・部品でほぼ可能（追加データ不要）
- ○ 軽い追加で可能
- △ データ追加（語彙・成り立ち等）が必要

活用できる既存アセット：
- `kanjiView(char)` … 読み（音/訓）・意味（KANJIDIC2 由来）
- `KanjiEntry.strokes` … 画数 ／ `KanjiEntry.level`・KANJIDIC2 grade … 学年区分
- `StrokeKanji.svelte` … 筆順アニメーション（書き順の再生）
- 達成型の山札・収集率（`deckTargetKanji` / `completedCount`）
- 図鑑（発見漢字の蓄積）／ `PlayStats`（プレイ履歴・ローカルのみ・送信なし）

## 1. 「作った瞬間」を学びに変える（プレイ画面）
- **◎ 学習カード化された巻物リビール**：合体成功時の巻物に、漢字＋**読み（音/訓）・意味・画数**を
  必ず表示する（`kanjiView` で取得）。演出を「気持ちいい」だけでなく「覚える」瞬間にする。
- **◎ 完成漢字の筆順再生**：成功時に `StrokeKanji` で**正しい書き順をなぞって再生**。
  作った字をそのまま書き順で確認できる。筆順データの無い字はフォント表示にフォールバック。
- **○ ミスを発見学習に**：ミス時に「実は ◯ と ◯ で △ が作れたよ」を控えめに提示（任意ON）。
  失敗を学びに変える。

## 2. 図鑑を「漢字学習帳」に格上げ
- **◎ 図鑑カードに学習情報**：読み・意味・画数・**部首名**を表示（部首は部品データから導出）。
  タップで筆順再生。
- **○ 学年別の収集率**：「小1: 78/80字 ✅ ／ 小3: 40/200字」のように**学年ごとの進捗バー**で
  ドリルの達成感を再現（grade 値で分類）。
- **○ 復習モード**：未収集・苦手な字だけを山札に集めて出す。`PlayStats` の履歴で
  「間違えた字」を優先する簡易の間隔反復（spaced repetition lite）。

## 3. 学年・カリキュラム連動（学校で使える鍵）
- **◎ 学年フィルタ**：やさしい（小学 1,006字）を**小1〜小6に細分**、ふつうを学年帯に。
  KANJIDIC2 の grade でそのまま分けられる。「小3だけ練習」が可能になり学校教材と並走できる。
- **○ 「今日の10字」ドリル**：学年から日替わりで対象を絞った短時間セッション
  （既存のデイリー＋山札を流用）。

## 4. 受動 → 能動（クイズ化）
- **○ 読みクイズ／意味当て**：作った直後に「この読みは？」を3択。または逆に
  **意味・読みを見せて部品から組ませる**（想起の強化）。
- **△ 熟語・例文**：「明 → 明るい／明日」などの用例。語彙データが現データセットに無いため、
  データ追加が必要（中〜将来）。

## 5. 定着・ふりかえり（保護者／先生向け）
- **○ 学習ログ**：練習字数・正答率・学年別達成を**ローカルのみで集計表示**
  （`PlayStats` に素地あり・送信なしでプライバシー安全）。

## 6. 追加要素候補（学習効果 × 実現性の高い順）

セクション1〜5を含む追加要素を、**学習効果と実現性の高い順**に一覧化（上ほど優先）。
実現性：◎ 既存データ・部品でほぼ可能／○ 軽い追加／△ データ追加・重実装が必要。

| 優先 | 要素 | 実現性 | 学習効果・狙い |
|---|---|---|---|
| 1 | **ふりがな表示トグル**（UI見出し・意味・説明） | ◎ | 低学年が**自力で遊べる前提条件**。無いと小学生が使えない |
| 2 | **段階ヒント**（①光る ②読み ③答え を選べる） | ◎ | 知識の壁を親切に吸収。既存ヒントの拡張。KPI「ヒント利用率」と整合 |
| 3 | **合体成功時の学習カード**（読み／意味／画数＋筆順再生） | ◎ | プレイ＝学習に直結。巻物＋`StrokeKanji` にデータを足すだけ |
| 4 | **読み上げ（音声）** | ○ | 「目＋耳」で定着。Web Speech API・送信なしで完結 |
| 5 | **にがて漢字リスト＆復習モード**（簡易SRS） | ○ | 間違えた/時間のかかった字を集中再出題＝**定着の核**。図鑑連動 |
| 6 | **学年別フィルタ（小1〜小6）＋図鑑の学年別収集率** | ○ | 学校教材と並走。grade で分類（要データ再生成） |
| 7 | **初回チュートリアル**（合体を1問だけ手取り足取り・スキップ可） | ○ | 離脱防止（PRD「触ればわかる」）。学習開始の前提 |
| 8 | **大きな文字・色覚配慮**（レアリティを色＋落款で示す） | ◎ | アクセシビリティ。既存の落款を活かせる |
| 9 | **実績・バッジ／学年マスター称号**（小3コンプ・100字達成 等） | ○ | 継続学習の動機。`PlayStats`＋localStorage |
| 10 | **連続プレイ日数（ストリーク）** | ○ | 毎日の学習習慣化 |
| 11 | **ミッション／お題**（魚へん3つ・画数10以上 等） | ○ | 部首・画数の学びを誘導 |
| 12 | **効果音・BGM（和風）**（オン/オフ設定） | ○ | 楽しさ＝学習継続。子ども向けに効果大 |
| 13 | **にがて漢字／図鑑のプリント出力** | ○ | 書き取りプリントとして家庭学習・宿題に使える実用機能 |
| 14 | **保護者／先生モード**（学年固定・1日上限・学習ログ閲覧） | ○ | 管理・見守り。広告ゼロは既に強み |
| 15 | **ハプティクス（合体時の振動）** | ○ | モバイルの触覚フィードバック（学習効果は補助的） |
| 16 | **部首名・成り立ちの一言解説** | △ | 「なぜこの部品で出来るか」。部首名は一部データから可、成り立ちは要追加 |
| 17 | **熟語・例文**（明→明るい/明日） | △ | 語彙学習に強いが、語彙データの追加が必要 |
| 18 | **書き取りトレース採点**（canvas になぞる） | △ | 書き順の体得に最も効くが重実装（`StrokeKanji` の発展） |

> 上位（1〜8）＝学習効果が高く既存資産で実現しやすい「まず入れたい」群。
> 中位（9〜15）＝継続・楽しさ・教材化の強化。
> 下位（16〜18）＝学習効果は高いがデータ追加・重実装が前提（中長期）。

## 着手順のおすすめ（直近の費用対効果）
1. **ふりがなトグル＋読み上げ** — 小学生が自力で遊べる前提条件。
2. **段階ヒント＋初回チュートリアル** — 知識の壁を親切に吸収。
3. **合体成功時の学習カード（読み／意味／画数＋筆順再生）** — プレイ＝学習に直結。
4. **学年別フィルタ（小1〜小6）** — 学校用途の核。
5. **にがて漢字の復習モード＋実績/ストリーク** — 「また来る理由」を学習文脈で作る。
