# T-025: E2E・品質ゲート仕上げ

| 項目 | 値 |
|---|---|
| フェーズ | Phase 5 |
| 優先度 | P0 |
| 依存 | T-014〜T-022 |
| 関連 | architecture 8 / development-guidelines 4 |

## 目的
主要シナリオのE2Eを整備し、CIの品質ゲートを完成させる。

## スコープ（実装する）
- Playwright シナリオ：基本プレイ（開始→10回→合体/ミス→終了）、詰み/手札0の両終了、図鑑収集率増加、デイリー再現性、オフライン
- CI仕上げ：`playwright install --with-deps`、ドメイン90%カバレッジ、`verify:data` ゲートの最終接続
- 主要KPI計測ログが出ることの確認

## 受け入れ条件
- [ ] 主要E2Eシナリオがすべて緑
- [ ] CIで lint/typecheck/test(90%)/gen:data/verify:data/build/e2e が順に通る
- [ ] いずれか失敗でマージ不可（ブランチ保護）

## スコープ外
- 新機能の追加

## テスト
- 本チケット自体がテスト整備

## 完了の定義 (DoD)
- [ ] 品質ゲートが完成し、MVPの回帰が守られる
