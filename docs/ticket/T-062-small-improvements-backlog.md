## 背景
develop レビューで挙がった、単独チケットにするほどではない小粒項目の受け皿。
着手時に 1 つずつ消化するか、近い変更のついでに拾う。

## 項目一覧

### コード整理
- [ ] `Settings.hintAlwaysOn` の削除（定義・migrate・既定値のみ存在し参照ゼロ。
      実装予定が復活するならチケット化して docs に紐付ける）。

### 文言・案内
- [ ] デイリーお題が **日本時間（JST）** で切り替わる旨を Home か About に一言案内。

### 運用・ビルド
- [ ] Yuji Syuku フォントサブセットの再生成をスクリプト化（UI 文言変更に追随できるよう
      対象文字列抽出 → サブセット生成を `scripts/` に用意）。
- [ ] 辞書 JSON（計約 520KB）のレベル別分割ロードの検討メモ化（現状は Brotli + プリキャッシュで
      問題なし。データ拡張時に再評価）。

### 記録（実装せず・方針メモ）
- [ ] CSP `style-src 'unsafe-inline'` は Svelte の動的 inline style 都合で恒久許容
      （nonce 移行は費用対効果が低い）。この判断を architecture.md に記録。
- [ ] SW 更新方式（autoUpdate + skipWaiting）はデプロイ頻度が上がったら
      `registerType: 'prompt'`（更新トースト方式）への切替を再検討。
- [ ] レアリティ識別は色＋落款ラベル併記を維持する（色のみに退化させない）旨を
      development-guidelines に一文追加。

## 受け入れ条件

  （全部一括でなく、消化した分だけ PR にしてよい）

## 完了の定義 (DoD)
