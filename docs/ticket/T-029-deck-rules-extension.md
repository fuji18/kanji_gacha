# T-029: 達成型ルール拡張（出題数選択・重複禁止・引き直しの山札返却）

## 目的
じっくり（達成型）に、出題数の選択・同じ漢字の重複禁止・引き直しの山札返却を導入する。
タイムアタックは対象外。

## スコープ（実装する）
- **出題数選択（2段フロー）**：モード選択 → 出題数選択（プリセット 5/10/20/50/全部）→ 開始。
  出題数 N＝対象から **N字をランダム選出し、その N字の部品だけで山札を構築**（`targetTotal=N`）。
  `DictionaryRepository.buildDeck` を「N字サンプリング＋RNG注入」に拡張。
- **重複禁止**：すでに作成済みの漢字を作る合体は **無効（ノーペナルティ）**。
  `SessionManager.combine` に「awarded が作成済みなら no-op・部品消費なし・コンボ維持」分岐。
  UI は「もう作ったよ」表示。
- **引き直しの山札返却**：`discardAndDraw` を「捨てた部品を山札へ戻してシャッフル→1枚引く」に変更
  （部品を場から失わない＝N字完成可能性を保つ）。
- **終了条件**：N字すべて完成でクリア／新たに作れる未完成字が無くなったら終了（収集率で評価）。

## 受け入れ条件
- [ ] 出題数をプリセットから選んで開始でき、山札が N字分で構成される
- [ ] 既出漢字の再作成は無効（ペナルティ・部品消費・コンボリセット無し）
- [ ] 「捨てて引き直す」で部品が山札に戻り、別の1枚が引かれる
- [ ] N字すべて完成でクリア、未完成到達不能で終了
- [ ] タイムアタックは不変
- [ ] lint / typecheck / unit / build / E2E すべて成功

## スコープ外
- レベル名称・学年分割（T-028）

## テスト
- 単体：buildDeck の N サンプリング、combine の重複無効、discardAndDraw の山札返却、終了判定
- E2E：出題数選択→N字完成でクリア

## 完了の定義 (DoD)
