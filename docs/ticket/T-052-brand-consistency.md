# T-052: ブランド一貫性（@fujioha/ui・@fujioha/tokens 取り込み）

| 項目 | 値 |
|---|---|
| フェーズ | Phase 8 |
| 優先度 | P2 |
| 依存 | T-049 |
| 関連 | fujioha_platform `packages/ui` / docs/ideas/fujioha-platform掲載スポーク作成プロンプト.md |

## 背景
playbook はスポークに `@fujioha/ui` / `@fujioha/tokens` の取り込みを**推奨**（必須ではない）。
当ゲームは独自の和風×Material トークンで構築済みのため、整合のとり方を検討する。

## 目的
プラットフォーム横断のブランド一貫性（配色トークン・基本UI）を、当ゲームの世界観を
損なわない範囲で確保する。

## スコープ（実装する・任意採用）
- `@fujioha/tokens` のベーストークンを取り込み、当ゲームの和風パレットを上書き層として整理する
  （ベース＝プラットフォーム、テーマ＝和風 の二層化）。
- 共通の「プラットフォームへ戻る」導線やフッタークレジットの要否を検討。
- 取り込みコストが見合わない場合は **採用見送り**を理由付きで記録（playbook 上は任意のため）。

## 受け入れ条件
- [ ] トークン取り込み or 見送り判断が記録されている
- [ ] 採用する場合、当ゲームの和風UIが視覚的に退行していない

## スコープ外
- ハブ/他スポークの変更

## テスト
- 視覚回帰（主要画面のスクリーンショット比較）

## 完了の定義 (DoD)
- [ ] ブランド一貫性の方針が確定し、必要なら反映済み
