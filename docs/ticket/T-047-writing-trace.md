# T-047: 書き取りトレース採点

| 項目 | 値 |
|---|---|
| フェーズ | Phase 7（レベル・学習拡張） |
| 優先度 | P3 |
| 依存 | StrokeKanji / 筆順データ拡充 |
| 関連 | docs/ideas/レベル設計と学習機能.md §6(18) |

## 目的
canvas に指/マウスでなぞって書き、書き順・形を採点して書き取り練習にする（最も学習効果が高い反面、重実装）。

## スコープ（実装する）
- お手本（`StrokeKanji` の筆順パス）に沿ってトレースする入力 UI。
- 採点：各画の方向・順序・許容誤差を判定（簡易）。
- 筆順データの拡充が前提（現状はデモ十数字）。

## 受け入れ条件
- [ ] お手本に沿ってなぞれ、書き順/形が簡易採点される
- [ ] 筆順データのある字で動作する

## 課題・前提
- 筆順データ（KanjiVG 等）の拡充とライセンス確認。採点アルゴリズムの調整が必要。

## 完了の定義 (DoD)
- [ ] 受け入れ条件を満たし、全品質ゲート成功
