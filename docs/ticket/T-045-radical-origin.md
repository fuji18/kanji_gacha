# T-045: 部首名・成り立ちの一言解説

| 項目 | 値 |
|---|---|
| フェーズ | Phase 7（レベル・学習拡張） |
| 優先度 | P3 |
| 依存 | データ追加 |
| 関連 | docs/ideas/レベル設計と学習機能.md §6(16) |

## 目的
「なぜこの部品で出来るか」を一言で示し、部首・成り立ちの理解を促す。

## スコープ（実装する）
- 部首名は部品データから導出（可能な範囲）。成り立ち一言は**データ追加が必要**
  （出典・ライセンス確認のうえ別データファイルで管理）。
- 学習カード／図鑑に表示。

## 受け入れ条件
- [ ] 部首名が表示される（対応字）
- [ ] 成り立ち解説データがある字で一言表示される
- [ ] データ未整備の字は非表示でフォールバック

## 課題・前提
- 成り立ちデータの出典確保とライセンス（CC 等）の確認が前提。

## 完了の定義 (DoD)
- [ ] 受け入れ条件を満たし、全品質ゲート成功
