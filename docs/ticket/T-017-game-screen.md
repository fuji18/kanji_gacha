# T-017: Game画面

| 項目 | 値 |
|---|---|
| フェーズ | Phase 3 |
| 優先度 | P0 |
| 依存 | T-015 |
| 関連 | PRD F1,F2,F4,F5 / 機能設計 6,7 |

## 目的
ガチャ・合体・救済の操作とゲーム状態表示を実装する。

## スコープ（実装する）
- `src/ui/screens/GameScreen.svelte` ＋ `components/`（`HandView`/`PartChip`/`GachaButton`/`ScoreBar`/`HintButton`）
- 手札表示・部品の複数選択・「合体！」・ガチャ・ヒント・捨てて引き直す
- スコア・コンボ倍率・ガチャ残の常時表示
- 手札上限到達時はガチャボタン非活性＋整理を促す
- 「選んで失敗」は `合体！` 押下時のみ判定（厳しくしない）

## 受け入れ条件
- [x] 部品を2つ以上選んで合体できる（成功/ミスの分岐）
- [x] 多部品選択が可能（強制しない）
- [x] ヒント/捨てるがレベル別コストで動く（SessionManager/RescueService 経由）
- [x] スコア・コンボ・ガチャ残が更新表示される
- [x] 手札上限でガチャが非活性化（実装済。現定数 GACHA_COUNT=10<HAND_CAP=12 では残0で先に非活性）

## スコープ外
- 演出エフェクト（T-018）／結果画面（T-019）

## テスト
- E2E：ガチャ→合体成功→スコア加算、ミス→コンボリセット

## 完了の定義 (DoD)
- [x] 1プレイがGame画面内で完結操作できる
