# T-027: タイムアタックモード

| 項目 | 値 |
|---|---|
| フェーズ | Phase 6（追加機能） |
| 優先度 | P1 |
| 依存 | T-010, T-014, T-016, T-017, T-019 |
| 関連 | 企画整理書 §11 / PRD F3,F4,F6 |

## 目的
運の比重が高い既存「じっくりモード（ガチャ残り回数制）」に対し、
**実力（速度・知識・連続性）が継続時間とスコアに直結する**並立モード
「タイムアタック」を追加する。既存モードは廃止しない。

## 背景
- ガチャ運に左右されにくい「実力優位」のゲーム性を求める要望（壁打ち 2026-06-13）。
- 詳細設計は企画整理書 §11「追補：タイムアタックモード」に確定済み。

## スコープ（実装する）

### ドメイン層
- タイムアタックの**時間延長計算**ロジック（純粋関数・TDD）
  - `延長秒数 = (基礎3秒 + 知識ボーナス + 速度ボーナス) × コンボ倍率`
  - 知識ボーナス：`floor(画数 / 4)` 秒、3部品以上の合体は `+2` 秒
  - 速度ボーナス：前回成功から3秒以内の連続正解で `+2` 秒
  - ミス：`-2` 秒（コンボリセットは既存ロジックを流用）
- 調整値は定数として一箇所に集約（`domain/constants.ts` 等）

### アプリ層（SessionManager）
- セッションに**モード種別**（`gachaCount` 既存 / `timeAttack` 新規）を追加
- タイムアタック時：
  - ガチャは**回数無制限・コストなし**で引ける
  - 持ち時間（初期30秒・上限なし）を保持し、成功で延長・ミスで減算
  - 終了条件は「残時間0」（手札0/ガチャ残の既存終了条件はタイムアタックでは不使用）
- 既存モードの挙動は不変であること（回帰なし）

### 永続化層
- **タイムアタック専用ベストスコアを別枠で保存**（既存 `bestScores` と混在させない）
- 図鑑・デイリーは両モード共通のまま

### UI層
- Home：モード選択（じっくり／タイムアタック）の導線を追加
- Game：残り時間バーを上部に常設（残5秒で朱色＋脈動、`prefers-reduced-motion` 時は色のみ）
- Game：タイムアタックではガチャ巻物演出を短縮またはスキップ可
- Result：モード別のベスト表示・新記録判定

## 受け入れ条件
- [ ] Homeからタイムアタックモードを選んでゲーム開始できる
- [ ] 初期持ち時間30秒で開始し、残時間が画面に常時表示される
- [ ] 合体成功で `(3 + floor(画数/4) + 速攻2) × コンボ倍率` 秒だけ時間が延長される
- [ ] 3部品以上の合体で知識ボーナスに +2秒 が加算される
- [ ] 前回成功から3秒以内の連続正解で速攻ボーナス +2秒 が加算される
- [ ] ミスで -2秒・コンボリセットされる
- [ ] 残時間0でゲーム終了し、結果画面へ遷移する
- [ ] タイムアタックのベストスコアが既存モードと別枠で保存・表示される
- [ ] 既存「じっくりモード」の挙動・既存E2E（31件）が無修正で通る
- [ ] `prefers-reduced-motion` 時はタイマーの脈動演出が無効化される

## スコープ外
- 調整値の本格バランス調整（プレイテストで実施）
- 初心者向け「ミス減時間0」練習サブ設定（将来検討）
- 極モード／対戦モード等の他追加モード

## テスト
- 単体：時間延長計算（基礎／知識／速度／コンボ乗算／ミス減）の境界値
- 単体：SessionManager のモード分岐・終了条件・ベスト別枠保存
- E2E：タイムアタック開始→成功で延長→ミスで減算→残0で結果遷移
- 回帰：既存31件のE2Eが無修正で通ること
